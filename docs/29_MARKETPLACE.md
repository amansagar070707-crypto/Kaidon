# Marketplace

## Purpose

Define what the Kaidon marketplace should become, later.

## Marketplace Objects

- agent templates
- plugins
- tools
- workflows
- UI components
- observability packs

## Marketplace Principles

- trust first
- versioned assets
- explicit permissions
- transparent provenance

## What To Delay

- payments
- broad public commerce
- complex reputation systems

## Why Delay

Marketplace before runtime maturity creates noise and low trust.

## Tradeoffs

- delaying marketplace slows ecosystem signaling
- it avoids building a distribution layer for unstable assets

## Alternatives

- early app-store style launch
- template gallery only

## Verified Facts

- Existing repo docs already describe marketplace as later-stage and secondary
  to runtime maturity.

## Assumptions

- The first useful marketplace is probably a curated registry, not an open
  bazaar.

## Speculative Ideas

- Kaidon marketplace assets may later carry runtime compatibility scores and
  security attestation badges.
