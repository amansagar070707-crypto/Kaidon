# Kaidon UI Implementation Spec

## Source

This spec is derived from the annotated Kaidon screenshot and the finalized brand rules.
It is the stricter implementation reference for product UI.

## Layout

- Dark base application shell.
- Left navigation rail with compact vertical items.
- Main content area with spacious top spacing and clear hierarchy.
- Header row with page title, search, and primary action.

## Surfaces

- Use the base, panel, and border tokens consistently.
- Keep panels slightly raised from the base background.
- Use thin borders and minimal shadows.
- Keep card density high but not crowded.

## Components

- Navigation rail
- Search field
- Primary CTA button in amber
- Agent cards
- Status pills
- Metadata rows
- Empty and loading states

## Interaction States

- Show running, idle, and muted states clearly.
- Use amber for active or primary actions.
- Use blue only for rare secondary data or links.
- Avoid mixed accent colors in the same element.

## Curves and Motion

- Use moderate radii and soft geometry throughout.
- Keep motion subtle and purposeful.
- Do not use flashy transitions or decorative animation.

## Copy and Density

- Keep labels short and technical.
- Favor information-rich cards over decorative hero sections.
- Avoid generic AI startup phrasing.

## Implementation Rule

- Any future UI should match this tone unless a new approved spec replaces it.
